06. Automatiza desde la terminal (CLI)

Objetivo

Aprender a usar OpenCode desde la línea de comandos para procesar documentos sin abrir la interfaz.

Duración estimada: 25 minutos

Consejo¿Por qué usar CLI?

El modo CLI es más rápido, preciso y escalable que el chat. Ideal cuando sabes exactamente qué quieres hacer y necesitas procesar varios archivos. Para más detalles, consulta la guía de ampliación: Ampliando CLI (HTML | PDF)

¿Qué es el modo CLI?

Además del chat interactivo, OpenCode puede ejecutarse con instrucciones directas desde la terminal. Esto permite automatizar tareas repetitivas.

El modo CLI es ideal para:

  • Procesar lotes — traducir, revisar o formatear varios archivos de una sola vez
  • Integrar con scripts — combinar OpenCode con otras herramientas del sistema
  • Automatizar en el servidor — ejecutar tareas programadas sin intervención manual

El comando básico es:

opencode run "tu instrucción"
NotaNota informativa

opencode sin argumentos abre la interfaz interactiva. opencode run ejecuta una instrucción y termina.

Conceptos clave

Concepto Qué significa
opencode run Ejecuta una instrucción en modo no interactivo
Entrada estándar Puedes pasar contenido por pipe (|)
Archivos Puedes referenciar archivos locales con @
Salida El resultado se imprime en la terminal

Manos a la obra

Ejercicio 1 — Explorar las opciones del CLI

Antes de ejecutar nada, mira qué opciones ofrece el CLI:

opencode --help
opencode run --help

El primer comando muestra las opciones globales. El segundo muestra los flags específicos de run, como --model o --temperature. Para ver los modelos disponibles en tu proveedor, consulta el paso 03.

Ejercicio 2 — Probar el comando básico

Abre una terminal en el directorio del curso y ejecuta:

opencode run "Dime tres beneficios de versionar documentos con Git"

OpenCode procesará la instrucción y te devolverá la respuesta en la terminal, sin abrir la interfaz gráfica.

Ejercicio 3 — Procesar un archivo completo

Pásale un archivo del curso para que lo revise:

opencode run "Revisa el estilo de este documento y sugiere mejoras" @paso-02.qmd

OpenCode leerá paso-02.qmd, lo analizará y te devolverá sus sugerencias directamente en la terminal.

Ejercicio 4 — Guardar resultado en archivo

Redirige la salida a un archivo para revisarla después:

opencode run "Resume este documento en 5 puntos" @paso-02.qmd > resumen.md

El resultado se guarda en resumen.md en lugar de mostrarse en la terminal. Usa > para sobrescribir o >> para añadir al final de un archivo existente.

Ejercicio 5 — Procesar varios archivos

Usa un bucle for para aplicar la misma instrucción a varios archivos:

for archivo in paso-01.qmd paso-02.qmd paso-03.qmd; do
  opencode run "Cuenta el número de secciones" @$archivo
done

Esto es lo que hace la CLI poderosa: la misma instrucción aplicada a múltiples archivos, sin repetir el proceso manualmente.

Ideas para tus propios comandos CLI

Comando Para qué sirve
opencode run "Revisa la ortografía" @archivo.qmd Corrección rápida
opencode run "Traduce al inglés" @documento.qmd > traduccion.qmd Traducción y guardado

opencode run "Extrae las referencias bibliográficas" @tesis.qmd Extracción de datos
