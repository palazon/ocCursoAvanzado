02. Skills reutilizables

Author

JA Palazón Ferrando

Published

May 30, 2026

Objetivo

Aprender a crear y usar skills —habilidades reutilizables— para tareas repetitivas de documentación.

Duración estimada: 25 minutos

¿Qué es un skill?

Un skill es un conjunto de instrucciones que le das a OpenCode para que se comporte de una manera determinada. A diferencia de los agentes, los skills:

  • No están siempre disponibles — se cargan bajo demanda
  • Son multiplataforma — valen para cualquier proyecto
  • Se guardan en tu carpeta de configuración global
  • Se invocan por nombre desde el chat

Mientras que un agente se define dentro de un proyecto (.opencode/agents/), un skill se define una sola vez y se usa desde cualquier carpeta.

Los skills son ideales para tareas que haces a menudo pero que no justifican un agente dedicado: formatear fechas, revisar tablas, aplicar una nomenclatura concreta, etc.

Cómo se crea un skill

Un skill se compone de dos partes:

  1. El archivo SKILL.md — con las instrucciones
  2. El registro en opencode.jsonc — para que OpenCode lo reconozca

Estructura de SKILL.md

# Nombre del skill

Describe aquí lo que hace el skill. Usa un tono directo, como
si le dieras instrucciones a un ayudante.

Registro en opencode.jsonc

{
  "skills": {
    "mi-skill": {
      "paths": ["/ruta/a/mi-skill"]
    }
  }
}

Manos a la obra

Ejercicio 1 — Crear la carpeta de skills

Los skills se guardan en ~/.config/opencode/skills/:

mkdir -p ~/.config/opencode/skills

Ejercicio 2 — Crear un skill de normalización de fechas

Crea la carpeta y el archivo del skill:

mkdir -p ~/.config/opencode/skills/formatear-fechas

Ahora crea ~/.config/opencode/skills/formatear-fechas/SKILL.md:

# formatear-fechas

Normaliza las fechas de un documento al formato corporativo
DD/MM/AAAA. Busca cualquier fecha en formato ISO (AAAA-MM-DD),
americano (MM/DD/AAAA) o abreviado (DD/MM/AA) y conviértela
al formato DD/MM/AAAA.

Regístralo en ~/.config/opencode/skills/skills.jsonc:

{
  "skills": {
    "formatear-fechas": {
      "paths": ["~/.config/opencode/skills/formatear-fechas"]
    }
  }
}

Si ya tienes un skills.jsonc, añade este bloque al array.

Ejercicio 3 — Probar el skill desde el chat

En Plan, prueba a cargar el skill:

Usa el skill formatear-fechas en el documento @informe.md

Si todo está bien configurado, OpenCode cargará las instrucciones del skill y las aplicará al documento que le pases.

Ejercicio 4 — Crear un skill de revisión de tablas

Crea ~/.config/opencode/skills/revisar-tablas/SKILL.md:

# revisar-tablas

Revisa todas las tablas del documento y verifica que:

1. Todas las tablas tienen un encabezado descriptivo
2. Las columnas están alineadas correctamente
3. No hay celdas vacías sin justificar
4. Los totales o resúmenes están en la fila correcta
5. El formato es coherente entre tablas similares

Señala cada problema encontrado, pero no modifiques el documento.

Añádelo al skills.jsonc:

{
  "skills": {
    "formatear-fechas": {
      "paths": ["~/.config/opencode/skills/formatear-fechas"]
    },
    "revisar-tablas": {
      "paths": ["~/.config/opencode/skills/revisar-tablas"]
    }
  }
}

Ejercicio 5 — Skills desde cualquier proyecto

Abre una terminal en otra carpeta cualquiera y prueba:

Usa el skill revisar-tablas

El skill se cargará aunque no estés en el proyecto del curso. Los skills son globales y están disponibles desde cualquier lugar.

Ideas para tus propios skills

Skill Para qué sirve
normalizar-glosario Unifica términos en un documento técnico
revisar-enlaces Comprueba que todos los enlaces están vivos
aplicar-tono-corporativo Ajusta el tono del documento al manual de estilo
extraer-citas Extrae todas las citas a un listado aparte
generar-indice Crea un índice de contenidos a partir de los encabezados
