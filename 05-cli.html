05. CLI — Automatizar procesos

Author

JA Palazón Ferrando

Published

May 30, 2026

Objetivo

Aprender a usar OpenCode desde la línea de comandos para procesar lotes de documentos sin abrir la interfaz.

Duración estimada: 25 minutos

¿Qué es el modo CLI?

Además del chat interactivo, OpenCode puede ejecutarse con instrucciones directas desde la terminal. Esto permite automatizar tareas repetitivas.

El modo CLI es ideal para:

  • Procesar lotes — traducir, revisar o formatear varios archivos de una sola vez
  • Integrar con scripts — combinar OpenCode con otras herramientas del sistema
  • Automatizar en el servidor — ejecutar tareas programadas sin intervención manual

El comando básico es:

opencode run "tu instrucción"

Conceptos clave

Concepto Qué significa
opencode run Ejecuta una instrucción en modo no interactivo
Entrada estándar Puedes pasar contenido por pipe (\|)
Archivos Puedes referenciar archivos locales con @
Salida El resultado se imprime en la terminal

Manos a la obra

Ejercicio 1 — Probar el comando básico

Abre una terminal en la carpeta del curso y ejecuta:

opencode run "Dime tres beneficios de versionar documentos con Git"

OpenCode procesará la instrucción y te devolverá la respuesta en la terminal, sin abrir la interfaz gráfica.

Ejercicio 2 — Procesar un archivo completo

Pásale un archivo del curso para que lo revise:

opencode run "Revisa el estilo de este documento y sugiere mejoras" @02-skills.qmd

OpenCode leerá 02-skills.qmd, lo analizará y te devolverá sus sugerencias directamente en la terminal.

Ejercicio 3 — Encadenar con tuberías (pipe)

Puedes pasar contenido por el tubo (|) desde otros comandos:

cat 03-git.qmd | opencode run "Resume este documento en 3 líneas"

Esto es útil cuando quieres procesar la salida de otros programas sin crear archivos temporales.

Ejercicio 4 — Guardar la salida en un archivo

Redirige la salida a un archivo para usarla después:

opencode run "Traduce este documento al inglés" @01-agentes.qmd > 01-agentes-en.md

El resultado se guarda en 01-agentes-en.md sin mostrarlo en pantalla.

Ejercicio 5 — Procesar varios archivos con un bucle

Usa un bucle del shell para aplicar la misma operación a varios archivos:

for f in *.qmd; do
  opencode run "Revisa la ortografía de este documento" @"$f" >> revisiones.txt
done

Este comando procesa todos los archivos .qmd del curso y acumula las revisiones en revisiones.txt.

Ejercicio 6 — Crear un script de automatización

Crea un archivo revisar-todo.sh en la raíz del proyecto:

#!/bin/bash
echo "=== Revisión automática del curso ===" > informe.txt
for f in *.qmd; do
  echo "--- Revisando $f ---" >> informe.txt
  opencode run "Revisa el tono y estilo de este documento" @"$f" >> informe.txt
  echo "" >> informe.txt
done
echo "=== Revisión completa ===" >> informe.txt

Hazlo ejecutable y pruébalo:

chmod +x revisar-todo.sh
./revisar-todo.sh

Abre informe.txt para ver los resultados de todos los archivos procesados.

Ejercicio 7 — Usar opencode run con agentes

Puedes invocar agentes personalizados desde CLI:

opencode run "@revisor-estilo Revisa este documento" @04-mcp.qmd

Siempre que el agente exista en .opencode/agents/, funcionará igual que desde el chat.

Buenas prácticas

  • Siempre revisa la salida — el CLI no es interactivo, no puedes corregir sobre la marcha
  • Usa archivos pequeños — para lotes grandes, procesa de uno en uno
  • Mide los tiempos — cada ejecución consume tokens; calcula el coste antes de lanzar procesos masivos
  • Combina con git — haz un commit antes de procesar, por si el resultado no es el esperado

Ideas para automatizar

Tarea Comando
Revisar ortografía de todos los .md for f in *.md; do opencode run "Revisa ortografía" @"$f"; done
Traducir un lote al inglés for f in *.qmd; do opencode run "Traduce al inglés" @"$f" > "${f%.qmd}-en.qmd"; done
