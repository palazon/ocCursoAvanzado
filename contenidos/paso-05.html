05. Conexión con sistemas externos (MCP)

Objetivo

Entender qué es MCP y por qué añadir contexto actualizado mejora las respuestas de OpenCode.

Duración estimada: 25 minutos

NotaNode.js es necesario

Los servidores MCP se instalan con npx, que requiere Node.js. Verifica que lo tienes con node --version antes de empezar.

¿Qué es MCP?

MCP (Model Context Protocol) permite a OpenCode acceder a información externa durante una conversación, sin necesidad de copiar y pegar manualmente.

Analogía: piensa en MCP como una biblioteca personal que OpenCode puede consultar cuando lo necesita. En lugar de darte la respuesta de memoria, busca en fuentes reales y actualizadas.

MCP fue creado por Anthropic en 2024 y donado a la Linux Foundation en 2025. Es un estándar abierto que funciona con múltiples herramientas: Claude, ChatGPT, Cursor, VS Code y otras.

Consejo¿Por qué importa el contexto actualizado?

Los modelos de IA tienen una fecha de corte de conocimiento. Por ejemplo, un modelo entrenado hasta 2024 no sabe lo que ha pasado en 2025. MCP resuelve esto: permite a OpenCode buscar información actualizada en fuentes reales, no solo usar lo que aprendió durante su entrenamiento.

Sin MCP: el modelo responde con lo que “recuerda” (puede estar desfasado).

Con MCP: el modelo busca en la fuente actualizada y responde con información reciente.

¿Para qué sirve?

Situación Sin MCP Con MCP
Consultar un glosario Copias y pegas el documento OpenCode lo lee directamente
Buscar en una wiki Abres el navegador y buscas OpenCode busca por ti
Consultar una base de datos Escribes la consulta SQL OpenCode la ejecuta

Ventaja clave: el contexto se actualiza automáticamente. Si alguien modifica el glosario, OpenCode siempre verá la versión más reciente.

Instalación: proyecto o global

Los servidores MCP se configuran en un archivo de configuración. Puedes instalarlos de dos formas:

Por proyecto (recomendado)

Crea .opencode/mcp.json en la raíz de tu proyecto:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/ruta"]
    }
  }
}

Global

Edita ~/.config/opencode/opencode.json para que esté disponible en todos tus proyectos.

Desde OpenCode (más fácil)

También puedes pedirle a OpenCode que instale y configure el servidor MCP por ti. Escribe en el chat:

Configura un servidor MCP de sistema de archivos para poder leer archivos de mi directorio de glosarios

OpenCode creará el archivo de configuración automáticamente.

Remoto o local

Los servidores MCP pueden ser:

Tipo Ejemplo Cuándo usarlo
Remoto GitHub, Notion, Brave Search Servicios en la nube
Local filesystem, sqlite Datos en tu ordenador
Desarrollado por ti Tu propio servidor Necesidades específicas

Los servidores locales son ideales para datos sensibles que no quieres subir a la nube. Los remotos son más fáciles de configurar y no requieren instalación.

Diversidad de modos de instalación

Los MCP se instalan de formas muy diferentes, dependiendo del tipo de servidor:

Modo Ejemplo Cuándo usarlo
npx (local) filesystem, sqlite Datos en tu ordenador
Remoto (URL) flashcards, context7 Servicios en la nube
Desde OpenCode cualquiera La más fácil
Con dependencias anki, docker Requieren software extra

Consejo: antes de instalar un MCP, es buena idea preguntar qué necesita. Algunos requieren API keys, otros software adicional (como Node.js o Docker). Preguntar antes ahorra tiempo y evita sorpresas.

Ejemplo rápido: un glosario consultable

Imagina que tienes un glosario de términos técnicos en tu equipo. Sin MCP, tendrías que copiar y pegar el documento cada vez que OpenCode necesite consultar algo. Con MCP, OpenCode puede leerlo directamente.

Ejercicio 1 — Crear el glosario

Crea un archivo glosario.md en un directorio de tu elección:

# Glosario

API: Interfaz de Programación de Aplicaciones
SLA: Acuerdo de Nivel de Servicio
MVP: Producto Mínimo Viable

Ejercicio 2 — Decirle a OpenCode que use el glosario

Escribe en el chat:

Configura un servidor MCP para que puedas leer archivos de el directorio donde tengo el glosario

OpenCode creará la configuración y podrás preguntarle:

¿Qué significa SLA según el glosario?

Y OpenCode responderá con la definición actualizada.

Ejemplo: flashcards de estudio

Los MCP no solo sirven para consultar información externa. También pueden ayudarte a aprender. Flashcards MCP permite crear tarjetas de estudio con repetición espaciada, directamente desde OpenCode.

Ejercicio 3 — Preguntar por prerequisites

Antes de instalar un MCP, es buena idea saber qué necesita. Escribe en Plan:

¿Qué necesita el MCP de flashcards para funcionar?

OpenCode te informará si necesita API keys, software adicional, o si es remoto (sin nada extra). En este caso, el MCP es remoto: solo necesita una URL.

Ejercicio 4 — Instalar desde OpenCode

La forma más fácil de instalar un MCP es pedirle a OpenCode que lo configure. Escribe en Plan:

Configura el MCP de flashcards para crear tarjetas de estudio

OpenCode creará la configuración automáticamente en opencode.json.

Ejercicio 5 — Crear flashcards

Una vez instalado, puedes usarlo para crear tarjetas:

Crea flashcards sobre los conceptos clave de MCP

OpenCode creará preguntas y respuestas, y usará repetición espaciada para recordarte cuándo repasar cada tarjeta.

Resumen del paso

  • MCP permite a OpenCode acceder a información externa
  • Añade contexto actualizado sin copiar y pegar
  • Se instala por proyecto, globalmente, o desde OpenCode
  • Puede ser remoto (URL) o local (con npx)
  • Antes de instalar, pregunta qué necesita el MCP
