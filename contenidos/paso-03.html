03. Modelos y proveedores de IA

Objetivo

Entender los modelos de IA disponibles en OpenCode, cómo alternar entre ellos y cómo conectar nuevos proveedores.

Duración estimada: 20 minutos

Modelos: ya tenemos opciones

Cuando instalas OpenCode, ya vienen configurados algunos modelos gratuitos (Zen). No necesitas registrar cuentas ni introducir claves API para empezar a trabajar:

  • Zen — modelos cloud gratuitos, vienen configurados por defecto
  • Locales — modelos que corren en tu ordenador sin conexión (Ollama, LM Studio)

Estos modelos son suficientes para la mayoría de tareas del curso. Si necesitas más calidad o velocidad, puedes conectar proveedores de pago (OpenAI, Anthropic, Google…) o usar OpenRouter como puerta de entrada a decenas de modelos.

Tres formas de acceder a un modelo

No confundir modelo de IA (el cerebro que responde) con modelo de acceso (la ruta por la que llega tu pregunta).

Acceso Analogía Ejemplos
Cloud Biblioteca pública: vas, consultas, vuelves Zen (gratuito), DeepSeek, GPT-4o-mini
Local Estantería propia: la tienes en casa Ollama, LM Studio (Qwen, Llama)
Desactivado Biblioteca cerrada por reformas Proveedor que apagaste
Consejo¿Cuál te conviene?
  • Empiezas → Cloud (sin configuración, gratis)
  • Privacidad → Local (todo se queda en tu red)
  • No usas → Desactivado (evita confusiones)

/models — Ver y cambiar de modelo

Escribe /models en el chat para ver los modelos disponibles y cambiar entre ellos:

/models

OpenCode te mostrará una lista con los modelos configurados. Selecciona uno y se usará en las siguientes conversaciones.

Qué modelo elegir

Modelo Tipo Mejor para
Zen (gratuito) Cloud Uso general, aprendizaje
GPT-4o-mini Cloud Revisiones rápidas
GPT-4o Cloud Documentos largos, complejidad
Claude Sonnet Cloud Análisis profundo, escritura

/connect — Conectar nuevos proveedores

Si los modelos por defecto no te alcanzan, puedes conectar proveedores adicionales con /connect:

/connect

OpenCode te guiará en el proceso. Necesitarás una clave de API del proveedor que elijas.

Proveedores clásicos

Proveedor Modelos habituales Variable de entorno
OpenAI GPT-4o, GPT-4o-mini, o1 OPENAI_API_KEY
Anthropic Claude Sonnet, Claude Haiku ANTHROPIC_API_KEY
Google Gemini Pro, Gemini Flash GOOGLE_API_KEY
Groq Llama 3, Mixtral (gratuito) GROQ_API_KEY

OpenRouter: puerta de entrada a múltiples modelos

OpenRouter es un proxy que da acceso a modelos de diferentes proveedores con una sola cuenta y una sola clave API. Es útil si quieres probar varios modelos sin gestionar múltiples suscripciones.

Ejercicio 4 — Conectar un proveedor (opcional)

Si quieres probar con otro proveedor:

  1. Ejecuta /connect y sigue las instrucciones
  2. Introduce tu clave de API
  3. Cambia al nuevo modelo con /models
  4. Prueba una tarea y compara
AvisoSeguridad: gestión de claves
  • Nunca versiones claves API — usa variables de entorno
  • Crea un archivo .env en la raíz del proyecto:
OPENAI_API_KEY=sk-proj-...
ANTHROPIC_API_KEY=sk-ant-...
  • Añade .env a .gitignore
  • OpenCode lee apiKey: "${VARIABLE}" y la resuelve del entorno
